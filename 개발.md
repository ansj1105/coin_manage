핵심 흐름 1. 입금
외부지갑
→ 유저 전용 입금주소(가상계좌)
→ 블록체인 입금 발생
→ 입금 감지 서버가 tx 확인
→ 어느 유저 것인지 식별
→ 내부 원장에 KORI 잔액 증가
→ sweep bot이 해당 주소 자산을 핫지갑으로 집금
핵심 흐름 2. 앱 내부 보유
유저 앱 화면
→ "내가 5,000 KORI 보유 중" 으로 표시
→ 실제로는 개인 온체인 지갑에 5,000 KORI가 있는 게 아님
→ 서비스 DB 원장에 5,000 KORI 권리값이 기록됨
핵심 흐름 3. 출금
유저 출금 요청
→ 내부 잔액 차감 또는 출금 보류
→ 리스크 검사 / 한도 검사
→ 핫지갑에서 실제 외부 주소로 KORI 전송
→ txHash 저장
→ 출금 완료
